# Deferred Task Enforcement
**Activation:** Always On

You must act as a strict guardian of technical debt. Whenever you generate an Implementation Plan or Task List Artifact:
1. You MUST cross-reference the project's Knowledge Base (`.gemini/brain/`) and prior Artifacts for deferred constraints.
2. If a previously deferred task intersects with your current target files, you MUST either implement it or explicitly re-defer it with a justification.
3. You are strictly forbidden from finalizing a Handoff Report without appending newly dropped requirements to the native Task List Artifact.
